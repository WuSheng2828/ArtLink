【这是啥】
随机词库是「随便来点」功能的后台。词库存放在 wordbank.json 里，
程序每次随机从里面挑词组合成一条提示词去生图。
v2.0 起使用「标签码」替代旧的 sfw/nsfw 分类，支持 36 个分类。

【文件结构】
项目根目录/
├── wordbank.json                      ← 最终词库（程序只读这个）
├── wordbank_filter.py                 ← 过滤规则框架（分类排序+对立检测）
├── wordbank_filter_config.json        ← 规则配置（关系开关+关系列表+标签码）
├── wordbank_parts_v2/                 ← 词库分块文件夹
│   ├── 01_基础信息类.txt ~ 10_艺术交互类.txt
│   ├── merge_wordbank_v2.py           ← 合并脚本
│   └── merge_wordbank_v2.bat          ← 双击合并
├── wordbank_editor.py                 ← 可视化编辑器主程序
├── wordbank_backups/                  ← 自动备份文件夹
├── 合并代码操作文件夹/wordbank_editor 文件分块/  ← 编辑器分块
└── run_editor.bat                     ← 启动编辑器

【标签码系统】
每条词不再只有 sfw/nsfw，而是携带多个标签码，例如：
  1girl → [sfw, 女性, 人类]
  fox ears → [sfw, 兽耳, 狐耳]
  nude → [nsfw, 全裸]
  penis → [nsfw, 男性, 生殖器]

SFW/NSFW 判断：看标签码中是否包含 sfw 或 nsfw。
全部标签码可在编辑器中查看和管理。

【操作流程】
1. 在 wordbank_parts_v2 里放 .txt 分块文件
2. 双击 merge_wordbank_v2.bat → 增量合并到 wordbank.json
3. 重启 ArtLink 或点击「刷新缓存」
4. 打开编辑器 http://127.0.0.1:19876 进行可视化编辑
5. 后台 http://127.0.0.1:5000 → 「随便来点」→ 开启开关测试

【系统概述】
随机词库是 ArtLink「随便来点」功能的支撑。v2.0 核心变化：
- 从 27 分类扩展为 36 分类（10 大类）
- 词条格式从 {text, zh, type} 改为 {text, zh, tags: [...]}
- 标签码体系替代 type 字段，支持多维度标记
- 过滤规则从硬编码改为配置文件驱动 + 标签关系引擎

【核心文件】
wordbank.json     — 词库数据（36分类，~1600条词）
wordbank_filter.py — 过滤规则（框架版：分类排序+对立检测+吸引权重）
wordbank_filter_config.json — 规则配置文件（排序/关系开关/关系列表/标签码全集）

【标签关系引擎】
配置位置: wordbank_filter_config.json → tag_relations 数组
关系类型:
  对立 — 两标签同时出现时触发重抽（强度: 弱/中/强）
  吸引 — 前面选了某标签后，后续相关标签加权（强×3 / 中×2 / 弱×1.5）
每条关系有独立 enabled 开关，可在编辑器中单独启停。

【TXT解析规则 v2】
- # 大类开头 → 跳过
- # id: 开头 → 跳过
- # 分类N：名称 (pick_min:X, pick_max:Y) → 新分类
- 支持 SFW/NSFW 不同取值格式: pick_min:0:sfw / 2~3:nsfw
- 数据行按逗号分割: text,zh,[tag1,tag2,...]
- 跳过含"指令""说明""readme"的非词库文件

【合并逻辑 v2】
merge_wordbank_v2.py 按文件名排序，增量合并。
已存在的词条（同分类+同text）跳过，不重复添加。
首次运行创建新词库，后续运行只追加新词条。
